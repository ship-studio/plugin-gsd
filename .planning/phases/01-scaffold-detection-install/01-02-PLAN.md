Create src/useGsd.ts implementing the full useGsd() hook. This is the core logic layer of the plugin.

Import React hooks from 'react' (they resolve to the host-provided React via Vite data-URL aliasing). Import useProject, useShell, useAppActions from './context'. Import PluginPhase, UseGsdReturn from './types'.

**Detection chain (detect function):**
The detect function walks the detection chain sequentially, setting phase state at each failure point and returning early. Order matters -- each check gates the next.

Step 1: Check if Ship Studio has a project open. If `project` is null, set phase to 'no-project' and return. CRITICAL: Never reference project.path without this guard -- TypeScript strict mode will catch it but the runtime will crash.

Step 2: Check GSD global installation. Use `shell.exec('bash', ['-c', 'test -d "$HOME/.claude/get-shit-done" && echo yes || echo no'])`. CRITICAL: Must use bash -c with $HOME -- tilde does NOT expand in direct shell.exec args. If stdout.trim() is not 'yes' or exit_code is not 0, set phase to 'gsd-not-installed' and return.

Step 3: Check project has .planning/ directory. Use `shell.exec('bash', ['-c', \`test -d "${project.path}/.planning" && echo yes || echo no\`])`. CRITICAL: Always use absolute path from project.path -- CWD is not guaranteed. If stdout.trim() is not 'yes', set phase to 'no-planning' and return.

Step 4: If all checks pass, set phase to 'has-planning'.

Wrap the entire chain in try/catch. On error, set error state and phase to 'error'. Set loading=true before the chain starts, loading=false in a finally block.

**Refs for unstable context values:**
Use the plugin-ralph pattern: store shell and actions in refs that update on every render, so useCallback functions have stable identity but always access current values.

```typescript
const shellRef = useRef(shell);
shellRef.current = shell;
const actionsRef = useRef(actions);
actionsRef.current = actions;
```

**detect() function:**
Wrap in useCallback with [project] as the only dependency (project determines the first guard check and the .planning/ path). Use shellRef.current and actionsRef.current inside the callback body.

**Initial detection (useEffect):**
Call detect() on mount and when project changes. Use a cancellation flag to prevent stale state updates:

```typescript
useEffect(() => {
  let cancelled = false;
  const run = async () => {
    await detect();
  };
  void run();
  return () => { cancelled = true; };
}, [detect]);
```

Note: The cancellation flag is set but not checked inside detect() in Phase 1 because detect() does not have intermediate state updates between async calls that would race. The flag prevents a potential double-fire. Phase 2 will add file-reading async calls where the cancellation check becomes essential.

**install() function:**
Wrap in useCallback with [detect] as the only dependency. Uses actionsRef.current and shellRef.current.

1. Set loading=true
2. Call `await actionsRef.current.openTerminal('npx', ['get-shit-done-cc@latest'], { title: 'Install GSD' })` -- this opens an interactive terminal. The Promise resolves when the terminal closes.
3. After the terminal closes, re-check the filesystem to determine if GSD was actually installed: `await shellRef.current.exec('bash', ['-c', 'test -d "$HOME/.claude/get-shit-done" && echo yes || echo no'])`
4. If stdout.trim() === 'yes': show success toast, then call detect() to update full state
5. If stdout.trim() !== 'yes': show error toast ('GSD installation was not completed')
6. Wrap in try/catch: on error, show error toast ('Failed to open installer terminal'), set error state
7. Set loading=false in finally block

CRITICAL: Do NOT assume install succeeded just because openTerminal was called. The user may have closed the terminal early (returns null). Always re-check the filesystem.

**Return value:**
Return `{ phase, loading, error, install, redetect: detect }` matching the UseGsdReturn interface.
