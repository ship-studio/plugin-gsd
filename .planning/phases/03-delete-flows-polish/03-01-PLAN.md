**src/useGsd.ts** — Add two new action functions before the return statement (after clearFileView). Follow the existing ref-stabilization and try/catch/finally patterns:

1. `deleteDirectory` — `useCallback(async () => { ... }, [project, detect])`:
   - Guard: `if (project === null) return;`
   - Call `shellRef.current.exec('bash', ['-c', \`rm -rf "${project.path}/.planning"\`])`
   - On `exit_code !== 0`: `actionsRef.current.showToast('Failed to delete .planning/ directory', 'error')` and return
   - On success: `actionsRef.current.showToast('Deleted .planning/ directory', 'success')` then `await detect()` — detect() will find no .planning/ and transition phase to 'no-planning'
   - Catch: `actionsRef.current.showToast('Failed to delete .planning/ directory', 'error')`

2. `deleteItem` — `useCallback(async (relativePath: string) => { ... }, [project, loadPlanning])`:
   - Guard: `if (project === null) return;`
   - VALIDATION: relativePath must start with `.planning/` — if not, `actionsRef.current.showToast('Invalid path', 'error')` and return
   - Call `shellRef.current.exec('bash', ['-c', \`rm -rf "${project.path}/${relativePath}"\`])`
   - On `exit_code !== 0`: `actionsRef.current.showToast(\`Failed to delete ${relativePath}\`, 'error')` and return
   - On success: `actionsRef.current.showToast(\`Deleted ${relativePath}\`, 'success')` then `await loadPlanning()` — refreshes dashboard without full redetect
   - Catch: `actionsRef.current.showToast(\`Failed to delete ${relativePath}\`, 'error')`
