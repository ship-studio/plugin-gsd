<task type="auto">
  <name>Task 1: Create lightweight markdown renderer</name>
  <files>src/utils/renderMarkdown.tsx</files>
  <action>
Create `src/utils/renderMarkdown.tsx`. This is a pure function: markdown string -> React.ReactNode. Uses regex-based line-by-line processing. NO external library — the Vite build externalizes React via data: URLs, so any bundled library adds to dist/index.js weight.

The renderer must handle this GSD-relevant markdown subset:
- `# H1`, `## H2`, `### H3` headings (styled with decreasing size/weight)
- `**bold text**` inline
- `` `inline code` `` inline (with background highlight)
- Fenced code blocks (``` ... ```) with `<pre><code>` — apply `overflow-x: auto` on the `<pre>` for long lines
- Unordered list items (`- item` or `* item`) — render with bullet character
- Ordered list items (`1. item`) — render with number
- Horizontal rules (`---`) — styled `<hr>`
- Blank lines — spacer div
- Plain paragraphs — `<p>` with inline markdown applied
- Markdown table rows (`| col | col |`) — render as simple styled rows (optional: skip if complex, show as plain text)

Implementation approach:
1. Split content by `\n`.
2. Iterate with a `while` loop (not `for` — fenced code blocks consume multiple lines).
3. For each line, test against patterns in priority order: fenced code block, H1, H2, H3, bullet list, numbered list, HR, blank, paragraph.
4. Use `createElement` and `Fragment` from React — import from 'react' (aliased in vite.config.ts).
5. Key each element with the line index (`key={i}`).

Inline markdown function `inlineMarkdown(text: string): React.ReactNode`:
- Split by `(/\*\*[^*]+\*\*|\`[^\`]+\`/g)`.
- Map parts: `**text**` -> `<strong>`, `` `text` `` -> `<code>` with inline style.
- Return array of React nodes.

Style all elements with inline styles using CSS variables:
- Headings: var(--text-primary) color, appropriate font-size/weight
- Code blocks: var(--bg-tertiary) background, monospace font, 11px size, border-radius: 6px, padding: 12px, `overflow-x: auto`
- Inline code: var(--bg-tertiary) background, 1px 4px padding, border-radius: 3px
- HR: var(--border) top border, 12px vertical margin

CRITICAL: The function must never throw. Wrap in try/catch returning a `<pre>` with raw content as fallback on any error.

Export: `export function renderMarkdown(content: string): React.ReactNode`

  <action>
**src/views/FileViewer.tsx** — Create new file. Purely presentational — receives UseGsdReturn as prop.

Props: `{ gsd: UseGsdReturn }`.

Component structure:

1. **Header row** (`.gsd-file-viewer-header`): horizontal flex with:
   - Back button: calls `gsd.clearFileView()`. Style with gsd-btn gsd-btn-secondary classes. Text: "Back" (or left arrow + "Back").
   - Breadcrumb: parse `gsd.activeFile` path to show "Phase N > filename". Example: `.planning/phases/01-scaffold-detection-install/01-01-PLAN.md` -> "Phase 1 > 01-01-PLAN.md". Extract phase number from directory name using `/^(\d+)-/` regex on the directory segment. If path doesn't match the phases pattern (e.g., `.planning/ROADMAP.md`), show just the filename.
   - Copy-path button: small icon/text button that copies `gsd.activeFile` to clipboard via `navigator.clipboard.writeText(gsd.activeFile)`. On success: `gsd.showToast('Path copied to clipboard', 'success')`. On failure: `gsd.showToast('Failed to copy path', 'error')`.

2. **Loading state**: if `gsd.fileLoading`, show "Loading file..." in .gsd-loading-indicator.

3. **Content body** (`.gsd-file-viewer-content`): if `gsd.fileContent` is not null, render `renderMarkdown(gsd.fileContent)`. This is a scrollable container — the modal body already has `overflow-y: auto`, and the file viewer content should flow within that.

4. **Error state**: if `!gsd.fileLoading && !gsd.fileContent && gsd.activeFile`, show "Could not load file" in .gsd-error-state.

Import `renderMarkdown` from `../utils/renderMarkdown`.
Import `UseGsdReturn` from `../types`.

CRITICAL: FileViewer MUST NOT call shell.exec or readFile. It only displays data from gsd.activeFile/gsd.fileContent. The readFile call happens in OverviewView when the user clicks a file.
