**src/views/OverviewView.tsx** — Add confirmState management, delete buttons, and ConfirmDialog rendering. Follow the existing pattern of keeping all interaction state local to the view component.

1. **Add import** at the top:
   ```typescript
   import { ConfirmDialog } from './ConfirmDialog';
   ```

2. **Add local state** after the expandedPhases state:
   ```typescript
   const [confirmState, setConfirmState] = useState<
     null | { type: 'full' } | { type: 'item'; path: string; label: string }
   >(null);
   const [isDeleting, setIsDeleting] = useState(false);
   ```

3. **Add handleConfirm callback** after the state declarations:
   ```typescript
   async function handleConfirm() {
     setIsDeleting(true);
     try {
       if (confirmState?.type === 'full') {
         await gsd.deleteDirectory();
       } else if (confirmState?.type === 'item') {
         await gsd.deleteItem(confirmState.path);
       }
       setConfirmState(null);
     } finally {
       setIsDeleting(false);
     }
   }
   ```

4. **If confirmState is not null, render ConfirmDialog INSTEAD of normal body content.** This is the inline replacement pattern from the research — avoids overflow/clipping issues. Wrap the entire existing return JSX in a conditional:
   ```typescript
   if (confirmState !== null) {
     return (
       <ConfirmDialog
         friction={confirmState.type === 'full' ? 'high' : 'low'}
         targetLabel={confirmState.type === 'full' ? '.planning/' : confirmState.label}
         onConfirm={handleConfirm}
         onCancel={() => setConfirmState(null)}
         isDeleting={isDeleting}
       />
     );
   }
   ```

5. **Add delete button to each phase row.** Inside the `.gsd-phase-row` div, after the plan count span, add a delete button. Use `e.stopPropagation()` so clicking the delete button doesn't toggle the accordion:
   ```tsx
   {phase.dirName && (
     <button
       className="gsd-delete-btn"
       title={`Delete ${phase.dirName}`}
       onClick={(e) => {
         e.stopPropagation();
         setConfirmState({
           type: 'item',
           path: `.planning/phases/${phase.dirName}`,
           label: phase.dirName,
         });
       }}
     >
       Delete
     </button>
   )}
   ```

6. **Add delete button to each file item.** Inside each `.gsd-file-item` div, after the file name text, add a delete button. Use `e.stopPropagation()` so clicking delete doesn't trigger readFile:
   ```tsx
   <button
     className="gsd-delete-btn"
     title={`Delete ${fileName}`}
     onClick={(e) => {
       e.stopPropagation();
       setConfirmState({
         type: 'item',
         path: `.planning/phases/${phase.dirName}/${fileName}`,
         label: fileName,
       });
     }}
   >
     Delete
   </button>
   ```

7. **Add "Delete all plans" button at the bottom**, after the phase list map:
   ```tsx
   {gsd.planningData.length > 0 && (
     <div className="gsd-delete-all-section">
       <button
         className="gsd-btn gsd-btn-danger"
         style={{ fontSize: 12 }}
         onClick={() => setConfirmState({ type: 'full' })}
       >
         Delete all plans
       </button>
     </div>
   )}
   ```

**Build:** Run `npm run build` to produce dist/index.js with all Phase 3 changes.

IMPORTANT: The delete buttons on phase rows and file items use `e.stopPropagation()` to prevent the click from bubbling to the parent onClick handlers (accordion toggle and readFile respectively). The ConfirmDialog replaces the body content when confirmState is not null — do NOT render it as a positioned overlay.
